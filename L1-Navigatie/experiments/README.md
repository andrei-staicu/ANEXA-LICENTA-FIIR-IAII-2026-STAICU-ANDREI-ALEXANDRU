# Experimental Data

This directory contains the experimental sessions used to evaluate the semantic-aware navigation system described in the accompanying manuscript. The experiments were collected on three physical robot platforms (Xplorer-A, Xplorer-B, Xplorer-C) executing the automated test protocol.

## Dataset Summary

| | Xplorer-A | Xplorer-B | Xplorer-C | Total |
|---|---|---|---|---|
| Architecture | Single RPi5 | Single RPi5 | Dual RPi5 | — |
| Sessions | 7 | 13 | 7 | 27 |
| Baseline legs (Test 2) | 10 | 19 | 16 | 45 |
| Adaptive legs (Test 3) | 22 | 34 | 14 | 70 |
| **Total legs** | **32** | **53** | **30** | **115** |

Overall success rate: 97% (111/115 validated). Baseline: 100% (45/45). Adaptive: 94% (66/70).

## Test Script

All sessions were generated by:

```
semantic_localizer/scripts/test_semantic_navigation_v5_10.py
```

See the main [README](../README.md) for usage instructions.

## Test Phases

| Phase | Description |
|---|---|
| `--test 0` (Setup) | Reset semantic map, reinitialize AMCL, regenerate Nav2-compatible graph |
| `--test 1` (ComputeRoute) | Single route planning request — validates Route Server functionality and route selection |
| `--test 2` (Baseline) | NavigateToPose using SmacPlannerHybrid + MPPI; no semantic information, no Route Server, no prescan |
| `--test 3` (Adaptive) | Semantic prescan (360° rotation + 35 s pipeline wait), ComputeRoute via Route Server, FollowPath with penalty monitoring and replanning (δ > 2.0) |

Each run in Tests 2 and 3 consists of a forward leg (start → goal) and a return leg (goal → start).


## Per-Run JSON Format

Each `test{2,3}_runXX_{forward,return}.json` file contains:

| Field | Description |
|---|---|
| `test_type` | `baseline_forward`, `baseline_return`, `adaptive_forward`, or `adaptive_return` |
| `execution` | Navigation outcome: `duration_s`, `distance_traveled_m`, `validated_success`, `avg_velocity_ms`, `max_velocity_ms`, `final_pose`, `trajectory_points` |
| `timing` | `prescan_duration_s` (adaptive only), `navigation_duration_s`, `total_duration_s` |
| `trajectory` | AMCL pose samples at ~2 Hz: `[x, y, timestamp]` |
| `trajectory_odom` | Odometry samples at ~5 Hz |
| `replanning` | (Adaptive only) `segments`, `total_reroutes`, `final_route`, `outcome` |
| `route_analysis` | (Adaptive only) `initial_route`, `final_route`, `initial_route_type`, `route_changed`, `planning_total_ms` |
| `semantic` | Detected objects count and classes at navigation end |
| `prescan_snapshot` | (Adaptive only) Objects detected during prescan |
| `min_object_distance` | Closest semantic object to the robot trajectory |
| `system_metrics` | CPU, memory, temperature samples at 1 Hz; YOLO diagnostics at 5 s intervals |

## Validation Criteria

A navigation leg is classified as **validated success** if all four conditions are met:

1. Nav2 action server reports success
2. Distance traveled ≥ 1.0 m
3. XY position error at goal ≤ 2.0 m
4. Navigation duration ≥ 3.0 s
